# Meeting Rooms II — Real-World Use Cases

## 1. Conference Room Booking Systems

The direct application: how many rooms does a company need given its meeting
schedule? Google Workspace and Microsoft 365 resource optimization runs exactly
this algorithm to recommend room configurations during office design.

## 2. Cloud Resource Provisioning

In cloud computing, how many VMs are needed at peak for a batch of jobs with
start/end times? The minimum number of VMs = maximum concurrent jobs at any
time = Meeting Rooms II. Used in AWS spot instance optimization and GCP batch
scheduling.

## 3. CPU Scheduling — Minimum Processors

In parallel computing, the minimum number of processors needed to run n
tasks with given start/end times is exactly Meeting Rooms II. Each "room"
is a processor; minimise processor count while completing all tasks.
